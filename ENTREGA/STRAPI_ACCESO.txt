ACCESO FUNCIONAL A STRAPI - LEARNIFY
====================================

INSTALACIÓN Y EJECUCIÓN LOCAL:

1. Clonar el repositorio:
   git clone https://github.com/Angelrp2/learnify.git
   cd learnify

2. Instalar dependencias de Strapi:
   cd strapi
   pnpm install
   (Si pnpm falla con errores SSL, usa: npm install)

3. Copiar archivo de configuración:
   cp .env.example .env

4. Iniciar Strapi en desarrollo:
   pnpm run dev

5. Acceder al panel de administración:
   http://localhost:1337/admin

PRIMERA EJECUCIÓN:

En la primera ejecución, Strapi te pedirá crear una cuenta de administrador.

Completa el formulario con:
- Email: tu email
- Contraseña: contraseña segura
- Nombre: tu nombre

DESPUÉS DE CREAR LA CUENTA:

1. Accede al panel de administración
2. En el menú izquierdo, verás las colecciones:
   - Categories
   - Instructors
   - Courses
   - Lessons
   - BlogPosts
   - ContactSubmissions

3. Crea contenido de prueba:
   - Agrega una categoría (ej: "Desarrollo Web")
   - Agrega un instructor (ej: "Juan Pérez")
   - Agrega un curso (ej: "Introducción a JavaScript")
   - Agrega lecciones al curso

ACCESO A LA API:

La API REST está disponible en:
http://localhost:1337/api/

Endpoints principales:
- GET /api/courses - Listar cursos
- GET /api/categories - Listar categorías
- GET /api/instructors - Listar instructores
- GET /api/blog-posts - Listar artículos

VERIFICACIÓN DE FUNCIONAMIENTO:

1. Inicia Strapi (paso 4 arriba)
2. Abre http://localhost:1337/admin
3. Crea contenido de prueba
4. En otra terminal, inicia Astro:
   cd astro
   pnpm install
   pnpm run dev

5. Abre http://localhost:3000
6. Deberías ver el contenido que creaste en Strapi

SOLUCIÓN DE PROBLEMAS:

Si Strapi no inicia:
- Verifica que tienes Node.js 18+
- Verifica que tienes pnpm instalado
- Elimina node_modules y pnpm-lock.yaml, luego reinstala

Si no puedes acceder a http://localhost:1337/admin:
- Espera 30 segundos a que Strapi termine de iniciar
- Verifica que no hay otro servicio en el puerto 1337

Si la API no devuelve datos:
- Verifica que creaste contenido en el panel de administración
- Verifica que los permisos públicos están habilitados

PERMISOS PÚBLICOS:

Todos los endpoints están configurados para acceso público (sin autenticación).
Esto permite que Astro acceda a los datos sin token.

Para cambiar permisos:
1. Ve a Settings > Roles > Public
2. Habilita/deshabilita acceso según necesites
