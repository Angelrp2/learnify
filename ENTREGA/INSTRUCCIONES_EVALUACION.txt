INSTRUCCIONES DE EVALUACIÓN - LEARNIFY
======================================

ESTRUCTURA DE LA ENTREGA:

1. MEMORIA_TECNICA.pdf
   - Documento principal con toda la información técnica
   - Incluye arquitectura, contenidos técnicos, estilos y SEO

2. GITHUB_REPOSITORY.txt
   - URL del repositorio público
   - Acceso a todo el código fuente

3. STRAPI_ACCESO.txt
   - Instrucciones para ejecutar Strapi localmente
   - Pasos de instalación y configuración

4. INSTRUCCIONES_EVALUACION.txt
   - Este archivo

CÓMO EVALUAR EL PROYECTO:

PASO 1: REVISAR LA MEMORIA TÉCNICA

Abre MEMORIA_TECNICA.pdf y revisa:
- Sección 3: Diseño y Tecnologías (arquitectura)
- Sección 3.1: Conceptos JavaScript (contenidos técnicos)
- Sección 4: Sistema de Estilos
- Sección 5: Estrategia SEO

PASO 2: ACCEDER AL REPOSITORIO GITHUB

Ve a https://github.com/Angelrp2/learnify

Revisa:
- README.md - Introducción al proyecto
- /docs/ - Documentación técnica detallada
- /strapi/ - Código del backend
- /astro/ - Código del frontend
- Historial de commits - Fases del desarrollo

PASO 3: EJECUTAR STRAPI LOCALMENTE

Sigue las instrucciones en STRAPI_ACCESO.txt:

1. git clone https://github.com/Angelrp2/learnify.git
2. cd learnify/strapi
3. pnpm install
4. cp .env.example .env
5. pnpm run dev
6. Accede a http://localhost:1337/admin

Crea datos de prueba (ver STRAPI_ACCESO.txt para ejemplos)

PASO 4: EJECUTAR ASTRO LOCALMENTE

En otra terminal:

1. cd learnify/astro
2. pnpm install
3. cp .env.example .env
4. pnpm run dev
5. Accede a http://localhost:3000

PASO 5: VERIFICAR FUNCIONALIDADES

En http://localhost:3000, verifica:

PÁGINA DE INICIO:
- Carga correctamente
- Muestra hero section
- Muestra cursos destacados

PÁGINA DE CURSOS (/cursos):
- Carga la lista de cursos
- Búsqueda funciona (escribe en el buscador)
- Filtros funcionan (selecciona dificultad o categoría)
- Paginación funciona (navega entre páginas)

PÁGINA INDIVIDUAL DE CURSO (/cursos/[slug]):
- Muestra detalles del curso
- Muestra lecciones
- Muestra instructor

PÁGINA DE BLOG (/blog):
- Carga artículos
- Muestra tiempo de lectura calculado automáticamente

PÁGINA DE RECURSOS (/recursos):
- Carga repositorios de GitHub en tiempo real
- Muestra información (estrellas, forks, issues)

PÁGINA DE CONTACTO (/contacto):
- Formulario valida campos
- Muestra errores si faltan datos
- Email valida con regex

DARK MODE:
- Haz clic en el botón de tema (arriba a la derecha)
- El sitio cambia a dark mode
- Los colores cambian correctamente

RESPONSIVE:
- Abre las herramientas de desarrollador (F12)
- Cambia a vista móvil (360px)
- El diseño se adapta correctamente

PASO 6: REVISAR CÓDIGO

En el repositorio, verifica:

/astro/src/lib/api.js:
- Fetch a Strapi con try/catch
- Manejo de errores

/astro/src/lib/utils.js:
- Búsqueda con regex
- Filtros dinámicos
- Paginación
- Validación de formularios

/astro/src/pages/cursos/index.astro:
- Componente de búsqueda y filtros
- Paginación

/astro/src/pages/contacto.astro:
- Validación de formulario
- Feedback visual
